# C-1.5: ネゴチャット（受諾前交渉）

> 反対提案後の交渉フェーズ。提案修正をやりとりしながら、双方合意で取引成立に進む。

## 画面の目的

C-1 受信で「反対提案する」が選ばれた後の、合意成立までの**チャット＋提案修正**画面。
- 提案内容（アイテム・待ち合わせ）の交渉
- メッセージのやりとり
- 7日期限・3日目/6日目リマインド・延長機能
- 双方合意で取引成立 → C-2 へ

### Props


### 前画面からの引き継ぎ

- `proposal_id`（C-1 receive または取引タブから）
- `proposal.status`：`negotiating` または `agreement_one_side`

### React state

メイン：チャット入力欄のテキスト、提案修正サブ画面の表示制御等。
状態の本体は `proposal_id` 紐付けで API から取得。

### データモデル参照

- `proposals` 1件（最新の提案内容＋ status, last_action_at, expires_at, agreed_by_*）
- `proposal_revisions`（提案修正履歴）
- `messages WHERE proposal_id = X ORDER BY created_at`
- 相手の `user_haves`（モーダル参照用、kind='for_trade'）

## アクション

### `C15NegoChatScreen` 主要操作

| 操作 | 動作 | 状態変化 |
|---|---|---|
| 提案カード「編集」ボタン | C-0 風編集モードへ | 提案修正画面へ遷移 |
| 「✓ 合意して受諾」 | M1 確認モーダル表示 | （未確定） |
| M1「合意する」 | API: agree → `agreement_one_side` または `agreed`（双方OKなら） | scenario が `mine-agreed` か `success` へ遷移 |
| 「📦 譲一覧を見る」 | M3 相手の譲モーダル表示 | （状態変化なし） |
| 「+ 希望に追加」（M3 内） | 提案修正でアイテムを追加 | proposals 更新 → system message 投稿 |
| メッセージ送信 | 入力テキストを `messages` に追加 | `messages.message_type='text'` で INSERT |
| + ボタン | アクションシート表示（写真添付・提案修正） ⚠️ | （状態変化なし） |
| 「⏱ 期限を +7日 延長」 | 期限を 7日延長 | `extension_count++`、`expires_at = expires_at + 7d` |
| 拒否ボタン | 確認モーダル → 拒否 | `proposals.status = 'rejected'` |

### 遷移

- 双方合意成立 → `C15-success` → C-2 へ
- 拒否 → 終了画面（取引タブ "過去取引"）
- 期限切れ → `C15-expired`（読み取り専用）

## バリデーション・エラー

### 合意送信のバリデーション

- 提案内容（アイテム・待ち合わせ）が完全に揃っていること
- ステータスが `negotiating` または `agreement_one_side`（`expired` / `rejected` では合意不可）
- 自分が直近に提案を修正していない or 修正後に相手が合意済みではない（⚠️ 要確認）

### メッセージ送信のバリデーション

- 入力欄が空でない
- 文字数上限（⚠️ 要確認、500-1000字程度想定）
- スパムフィルタ・禁止ワード（⚠️ 要確認）

### 提案修正時のバリデーション

C-0 と同等のバリデーション（提示物 + 待ち合わせ）。

⚠️ 要確認：提案修正で `last_action_at` が更新され、7日カウントがリセットされるか（09 §1 未確定項目#1）

## エッジケース

| ケース | 挙動 |
|---|---|
| 期限切れ瞬間にアクション中 | サーバ判定で reject、エラーメッセージ「期限切れです」 |
| 双方が同時に「合意送信」 | 後勝ち or 楽観ロック。サーバで `agreed_by_*` を atomic 更新 |
| 提案修正中に相手が合意送信 | 自分の修正反映後、相手の合意は無効化（`agreed_by_them=false`） ⚠️ 要確認 |
| メッセージ取得が失敗 | エラー表示 + リロードボタン |
| ネットワーク切断中の送信 | キューに保持、復帰時に送信 ⚠️ 要確認 |
| sender が rejected_partners に追加されてた | この画面に来ない（C-1 receive で弾かれる） |
| 同じ提案で disputes が作られている（dispute中） | このネゴは凍結、通常メッセージ送信不可 ⚠️ 要確認 |

## 状態遷移（09 との対応）

`09_state_machines.md` §1 Proposal Lifecycle：

```
negotiating → negotiating（提案修正、メッセージ送信）
negotiating → agreement_one_side（一方が合意送信）
agreement_one_side → agreed（もう片方も合意送信）
agreement_one_side → negotiating（提案修正で取り下げ）
negotiating → rejected
negotiating → expired（7日経過）
```

合意成立後は `09 §2 Deal Lifecycle` の `agreed → on_the_way → ...` へ。
