# システム要件定義（随時更新）

## 戦略コンテキスト（詳細は 03_strategy.md）

- **MVP範囲**：現地交換に特化（郵送はYahooフリマと棲み分け）
- **取引対象**：ランダム封入の小型グッズ
- **対象シーン**：場所を問わない
- **取引形態**：1:1中心
- **既存SNS連携**：Xを捨てない、双方向の導線
- **ジャンル戦略**：横断スタート

## 機能要件

### F1. 在庫登録（自宅在庫）
- グッズマスタ参照（**キャラは運営マスタ管理**）／新規追加
- **状態タグ**（美品／コーティング有／シリアル付き等）の標準化
- **数量入力（×N）**：だぶり登録対応
- **写真撮影UX**（プロトタイプ準拠）：
  - 複数グッズを1枚撮影 → **手動切り抜き**（MobileSAM・背景白塗りは不要）
  - 400×400 JPEG
  - 切り抜き各グッズに **キャラ名・グッズ種別** を割当
  - **グッズ種別の最低限の自動推定**（あれば嬉しい・必須ではない）
- **公開モデル**：登録すれば全公開（プライベート在庫・可視性レベル・交換ポリシータグはMVPで廃止）

### F2. 携帯グッズ（現地モード）※命名は暫定
- 「今日持参する」フラグで自宅在庫から絞り込み
- Availability Window（時間枠＋場所）との紐づけ
- **開封直後の即時登録UX**（時間プレッシャー対応）：
  - 切り抜きスキップ可（後で整える）
  - ワンタップ撮影 → アップロード → マッチ表示

### F3. 求めるグッズ登録（ウィッシュリスト）
- **キャラ運営マスタ参照**（マッチ精度のため）
- **優先度の表現**：最優先／2番手／妥協OK
- 妥協アイテムは通知対象外、検索時のみヒット

### F4. マッチング（v2 — iter61.11 で更新、詳細は notes/18_matching_v2_design.md）

**3 軸のマッチング構成**:
- **広域マッチ**（global match、ホームのデフォルト）
  - 時空制限なし、譲 × wish のみで全国マッチ
  - 完全マッチ / 片方向マッチ（forward / backward）の 2 種は維持
- **現地マッチ**（local match、ホーム上部の pill で切替）
  - AW（場所×時間×半径）に時空交差した相手のみ表示
  - 携帯グッズ（carrying=true な譲）と選択した wish のみ照合
  - 位置情報はモード ON 時に GPS で毎回更新、それ以外の設定（半径・選択 wish 等）は永続化
  - 持参グッズと wish の **一括リセット**ボタン提供
- **個別募集**（pinpoint listing、UI 表記は「個別募集」）
  - 譲 1 + wish 1 + 比率（1:1〜10:1）+ 優先度（1〜5）+ タグ
  - 既存 wish から「これで個別募集する」流れで作成
  - 通常マッチ結果に上乗せして優先度順で先頭表示

**基本データモデル**: Availability Window は変更なし（場所＋許容半径＋時間範囲＋メモ）

**イベント情報はタグ**（時間・場所を一括入力する UX ショートカット）

**UI タブ構成**: [ 完全マッチ ] [ あなたを求めてる人 ] [ あなたが求めてる人 ] [ 探索 ]
- 広域 / 現地のどちらのモードでも有効

**「同種交換 / 異種交換 / 混合交換」のタグ**:
- wish と個別募集に付与する **自己申告タグ**
- マッチング演算では **判定に使わない**（弾かない）
- マッチング結果カードに chip 表示し、ユーザーが目視で「自分の譲が当てはまるか」判断

**打診プロセス**:
- 両者に半マッチ通知 → どちらかが打診ボタン → 自動生成テンプレ＋編集可 → 相手が諾否 → OK でチャット開始
- **打診時にカレンダー公開オプション**（任意・default OFF）
  - ON で相手と自分の AW を時系列で重ね見できる（calendar overlay UI）
  - 取引完了で自動非公開

**「もう持ってる／断った」学習**（MVPに含む）：
- 「自分も持ってる」マークで以後除外
- 「断った」を片方向で記憶
- NO 時の定型文選択肢

**片方向 → 完全マッチ昇格導線**（採用検討中）：
- 個別募集が代替手段として機能する可能性
- 相手プロフから相手の譲一覧を見て、興味あるものを探す

**通知制御**: **MVP では考えない**（後続フェーズで再設計）

**L4（3者交換）はMVP対象外**

### F5. コミュニケーション
- iHub内DM／取引チャット
- 場所・時間の合意（地図／会場ピン）
- **Xへの投稿テキスト・画像の自動生成**（在庫・ウィッシュ選択 → 譲求テキスト＋写真）
- **Xからの「iHubで取引する」誘導リンク**
- **マッチ前のプロフ表示**：性別、評価サマリ（★4.8／取引127回／無断キャンセル0回）
- **マッチ後のプロフ表示**：取引時の格好写真（直前更新可）、交換物詳細

### F6. 物理交換（現地）【最重要】
- **合流支援**：目印登録／QRコード提示／会場マップ／**当日昼〜夕の服装写真直前更新**
- **交換証跡：両者の交換物を1枚の写真に収める設計**【最重要】
  - 通常の写真アップロード（透かし・別解像度プロファイルは不要）
  - 撮影画像にメタデータ自動付与（両者ID／タイムスタンプ／任意で位置）
  - 両者の相互承認で取引完了
  - 写真は両者のアカウントに証跡として保存
- 安全性確認：性別公開、年齢任意開示
- **オフラインファースト【最重要】**
  - 在庫・ウィッシュ・既存マッチ・相手連絡先・合流場所はローカルキャッシュ
  - 証跡撮影はオフラインで撮影、ネット復帰時に自動アップロード
  - 新規マッチング演算のみオンライン依存

### F7. 信頼・安全性
- **評価／取引実績の集約**
  - サマリ数値：合計★／取引回数／無断キャンセル数
  - マッチ前から確認可能（誹謗中傷化リスクのため詳細コメント表示は要検討）
- 詐欺通報・ブロック
- 鍵垢相当のプライバシー設定（マッチ相手以外プロフ非表示）
- **未成年保護**：年齢任意開示・現地合流時の保護者同伴推奨
- 性別自己申告（信頼性は通報フローで担保）

### F8. イベント情報
- ユーザー作成イベントタグ
- **既存タグへのサジェスト・名寄せ**（重複防止）
- 公演／物販イベントの時間・場所を一括入力
- 公式情報の自動取り込みは**後フェーズ**

### F9. キャラ／グッズ運営マスタ
- **キャラは運営側で管理**（検索・マッチ精度のため）
- ジャンル → グループ → メンバー の階層
- 表記揺れ吸収（別名・英表記・略称）
- グッズ種別の標準カテゴリ（トレカ／生写真／缶バッジ／アクスタ等）

## 非機能要件

- **時間プレッシャーへの応答性**：会場で1〜2分以内に交換成立まで進める操作量
- **オフライン耐性**：会場の電波が悪い前提（最重要）
- **プライバシー**：本名不要・現地合流時の匿名性維持／マッチ相手以外への情報非開示徹底
- **Mobile First**：会場での片手操作前提
- **アクセシビリティ**：人混み・屋外・夜間の操作性

## 重要な設計論点（議論中）

- **片方向マッチ→完全マッチ昇格導線**：採用するかシンプルな半マッチに留めるか
- **キャラ運営マスタの初期範囲**：どのジャンルからカバーするか
- **位置情報の扱い**：場所・時間検索／GPS精度／プライバシー
- **公式との関係**：交換禁止公演の扱い
